# THE UNIVERSITY
> The source of truth for all knowledge gathered on the journey of learning and building with AI.

## What This Is

The University is a chronicle — a forge where raw knowledge becomes understood knowledge. Every course, tutorial, article, and discovery is captured, processed, and forged into something that can be revisited and built upon.

This is not a course tracker. This is a knowledge system.

## How It Works

Three layers for every entry:

1. **RAW** — verbatim source material. Never edited. Always sourcable.
2. **FORGED** — synthesized understanding. Where learning becomes your own.
3. **META** — the machinery: intakes, progress logs, chapter indexes.

## Entry Types

| Prefix | What It Is | Example |
|--------|-----------|---------|
| `COURSE_` | Formal course or certification | `COURSE_001_ai-fluency` |
| `LEARNING_` | Informal learning: articles, videos, tutorials | `LEARNING_001_python-async` |
| `RESOURCE_` | Curated references, tools, frameworks | `RESOURCE_001_prompt-patterns` |

## Directory Structure

```
university/
├── README.md              ← this file
├── CONSTITUTION.md        ← rules, formats, conventions
├── REGISTRY.md            ← index of all entries
├── CHRONICLES.md          ← dated narrative of the journey
├── meta/
│   ├── intakes/           ← application targets, learning goals
│   ├── frameworks/        ← reusable mental models
│   ├── patterns/          ← discovered patterns
│   └── glossary/          ← terminology collected along the way
├── courses/               ← all COURSE_ entries
├── learning/              ← all LEARNING_ entries
├── resources/             ← all RESOURCE_ entries
└── templates/             ← starter templates for new entries
```

## Navigation

- **New entry?** Check `CONSTITUTION.md` for naming and format rules
- **Lost?** Check `REGISTRY.md` — everything is indexed there
- **Want context?** Check `CHRONICLES.md` — the story of how we got here

## Principles

1. RAW is scripture — never modify captured source material
2. Commits tell the story — every commit documents what, when, why, from where
3. Forged beats collected — understanding > hoarding
4. The system grows with you — adapt the structure as needs change
